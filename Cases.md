# Конфигурация проекта:
## Режим работы(Для RT сборок)
1) RT1 - запуск действия на каждую ноду при её активации(Оптимально по ресурсам - медленно по скорости)
2) PRT - запуск нод, как сервисов, ожидающих входные данные (Оптимально по скорости на линейных процессах, где нет повторяющихся нод. Средне по ресурсам)
3) PRTf - запуск нод, как сервисов, по отдельному реализатору на каждую ноду (Имба скорость, залупа ресурсы)

## Настройка RT сборок
По цели:
1) Linear - сборка, сконфигурированная для выполнения задач одного цикла. Идеально подходит для компиляции, обработок нейросетей и билд/парс скриптов. 
Поддерживает ивенты, однако пайплайн будет завершён при выполнении всех нод класса target. Поддерживает сохранение сессии при ошибке.
Ограничения:
- По сути нет, однако нежелательно пихать ивенты в качестве стартер нод.
2) Service - сборка, сконфигурированная для оркестрации сервисов. Будет работать, пока не прервано. Прекрасный выбор для сборки локальных сервисов и их формализации.
Ограничения:
- У одного таргета может быть только 1 ивент. По сути обходиться через события функции и глобальные переменные, но это необходимая формальность.
