# Фронт

На нодах реализуется пайплайн. 
Ноды берутся из паков.(Паки описаны позже)
Фронт запрашивает доступные паки у butlerа.

# Butler

Этакий дворецкий. Фронт связывается с ядром не напрямую, а через него. Делается это, чтобы разгрузить входящие данные на ядро, и отдать часть задач по связи с клиентом.

# Ядро

Когда паплайн на фронте готов, он отправляется ядру. Ядро собирает из пайплайна C request, 
который передаётся сборщику, либо интропретатору(В зависимости от режима).



# Реализаторы
## Сборщик
Если при отправке пайплайна указан режим compI :-> сборщик собирает по нуждам пайплайна docker.iso, 
и отсылает его, в качестве воркера на сервер, либо клиенту, в зависимости от расширения у compI(runit/return)


## Интропретатор
Получает C request и начинает исполнять его в реальном времени, с синхронизацией визуалов с Фронтом. 
В основном имеет смысл использовать на задачах, в которых долго выполняются конкретные ноды, и минимально циклов на нодах.

Преимущества: видишь прогресс в реальном времени, легче ориентироваться при ошибках, быстрее компилируется.
Недостатки: убогая производительность на сложных задачах.
